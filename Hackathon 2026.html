1
Día 1
Kick-off & Diseño
Presentación de reglas, criterios y rúbrica completa
Cada equipo propone su idea y la valida (10 min + feedback abierto)
Definen arquitectura, endpoints y wireframes básicos
Crean el repositorio — ambos integrantes hacen su primer commit
2
Día 2
Construcción core
Sprint de desarrollo enfocado en funcionalidad principal
Check-in de mentoría al mediodía (15 min por equipo) — sin juicio de puntaje
Al final del día: mínimo 4 endpoints funcionales en el backend
La mentoría es un espacio de apoyo, no de evaluación.
3
Día 3
Integración & Polish
Frontend conecta con el backend real — sin mocks
Al menos un flujo completo de punta a punta
Se escribe el README y documentación mínima
Ensayo interno de la demo (máx. 12 min)
4
Día 4
Demo Day
Freeze de código al mediodía (12:00 PM) — el último commit antes de esa hora es el evaluado
30 minutos por equipo: 22 de demo + 8 de preguntas
La demo debe mostrar el flujo principal, una llamada real al backend y al menos un manejo de error
Code freeze: 12:00 PM
El lead verifica el timestamp en Git
Entregables obligatorios

01

GitHub / GitLab Obligatorio

Markdown Obligatorio
03
Demo en vivo
Código corriendo — no slides. Flujo principal + backend real + manejo de error.
Código en vivo Obligatorio
Reglas de participación

Lo que está dentro y lo que no.

Solo código nuevo — no reutilizar proyectos anteriores de la empresa ni personales
Stack obligatorio: Spring Boot (back) + React + TypeScript (front)
Ambos integrantes deben tener commits en el repositorio
APIs externas permitidas si son gratuitas o de prueba; no de pago sin aprobación
Cualquier librería open source y framework de UI está permitida
Un juez externo a los equipos evalúa (lead o manager del área)
IA permitida (Copilot, ChatGPT) — el código debe ser comprendido y explicable
El plagio descalifica al equipo — todo el código debe ser original del hackathon
